## 2. Abstract

Modern software projects often suffer from poor documentation, making code understanding difficult for developers, students, and maintainers. The **Code Documentation Navigator** solves this problem by using Artificial Intelligence and Retrieval-Augmented Generation (RAG) to automatically analyze source code and generate human-readable explanations.

The system allows users to upload a ZIP file of any project. It extracts the source code, processes it using semantic embeddings, stores it in a vector database, and enables users to ask natural language questions. The system retrieves the most relevant code sections and generates clear, understandable explanations using a free open-source language model.

This project demonstrates the practical application of AI in software engineering, documentation automation, and developer productivity tools.

## 8. System Modules

### 8.1 ZIP Upload Module

* Accepts project ZIP file
* Extracts files
* Creates workspace directory

### 8.2 Code Loader Module

* Reads source code files
* Supports multiple file formats
* Filters valid programming files

### 8.3 Text Splitter Module

* Breaks large files into chunks
* Prevents context overflow
* Optimizes retrieval

### 8.4 Embedding Module

* Converts code to vectors
* Semantic representation
* Uses MiniLM model

### 8.5 Vector Database Module

* Stores embeddings
* Enables similarity search
* Persistent storage using ChromaDB

### 8.6 Retrieval Module

* Finds relevant code chunks
* Semantic matching

### 8.7 LLM Module

* Uses Flan-T5 model
* Generates explanations
* Converts code context to human language

### 8.8 UI Module

* Streamlit web interface
* File upload
* Question input
* Answer display
* Code preview

## 9. Workflow

1. User uploads ZIP project
2. System extracts files
3. Code files are loaded
4. Code is split into chunks
5. Embeddings are generated
6. Stored in vector database
7. User asks question
8. Relevant code retrieved
9. Context sent to LLM
10. LLM generates explanation
11. Explanation shown to user
